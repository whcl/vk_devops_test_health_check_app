### Описание решения
Для написания веб-приложения я использовал FastApi, c реализацией можно ознакомиться в директории fastApiProject. 
<p> Скрипт мониторинга написан на Python, с реализацией можно ознакомиться в директории HealthChecker. Скрипт мониторинга периодически делает get запрос на endpoint веб-приложения. 
<p> Если приложение упало, то скрипт мониторинга вызывает метод перезапуска, который с помощью docker.socket делает перезапуск контейнера по его имени. 
<p> Для автоматизации использованы docker-compose и Makefile. Благодаря docker-compose достигается возможность запускать скрипт мониторинга всегда при запуске приложения. 
<p> Также благодаря одной сети удобно обращаться по имени контейнера к веб-приложению. Makefile позволяет запускать всю систему с интервалом мониторинга по умолчанию, либо задавать свое значение.
<p> Пример работы в images/example.png. Я оставил в Makefile команды без фонового режима, чтоб было видно логирование.

### Установка и запуск

```bash
# Запуск с интервалом проверки 5 секунд (по умолчанию)
make install

# Запуск с кастомным интервалом
make install INTERVAL=10
